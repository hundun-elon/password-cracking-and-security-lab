Section 1: Hashing (Without Libraries)

SIMPLE

ASCII Sum Hasher

Scenario: You are building your own basic hash function for a top-secret club. Instead of using any Python libraries, your job is to write a function my_hash(word) that adds the ASCII values of each character in the word and returns the total as the hash.
Example Test Cases:
Input: "cat" → ASCII: c=99, a=97, t=116 ⇒ Hash: 312
Input: "dog" → ASCII: d=100, o=111, g=103 ⇒ Hash: 314
Input: "hat" → ASCII: h=104, a=97, t=116 ⇒ Hash: 317
Task: Write the function and test it on at least 3 different words of your choice.
MEDIUM

Shifted ASCII Hasher

Scenario: A mysterious hacker encrypts usernames by shifting each letter by +1 in ASCII (e.g., a → b), and then adds the ASCII values of the shifted characters.
Function Specification:
  • Define shifted_hash(word).
  • For each character, shift it by +1 in ASCII.
  • Return the sum of ASCII values of the shifted characters.
Example Test Cases:
Input: "abc" → "bcd" = 98 + 99 + 100 = 297
Input: "root" → "sppu" = 115 + 112 + 112 + 117 = 456
Input: "admin" → "benjo" = 98 + 101 + 110 + 106 + 111 = 526
Task: Try this hash on usernames like "admin", "guest", and "root".
HARD

Custom Hash Function Challenge

Scenario: A security company needs a more robust hash function that reduces collisions (when different inputs produce the same hash value). They need a function that incorporates not just the ASCII values, but also the position of characters.
Function Specification:
  • Define position_hash(word).
  • For each character, multiply its ASCII value by its position in the string (starting from 1).
  • Sum these values and return the result modulo 1000 (to keep the hash value within a reasonable range).
Example Test Cases:
Input: "abc" → (97×1) + (98×2) + (99×3) = 97 + 196 + 297 = 590
Input: "cba" → (99×1) + (98×2) + (97×3) = 99 + 196 + 291 = 586
Input: "password" → ((112×1) + (97×2) + (115×3) + ... + (100×8)) % 1000 = 766
Task: Implement this function and test it with at least 5 different words. Discuss why this might be more collision-resistant than a simple ASCII sum.

Section 2: Ciphers

SIMPLE

Caesar Cipher

Scenario: A spy is sending secret messages using the Caesar Cipher with a shift of +3. Your task is to write a function that can both encrypt and decrypt a message using this cipher.
Function Specification:
  • Define caesar_encrypt(text, shift) and caesar_decrypt(text, shift).
  • Shift letters forward (encrypt) or backward (decrypt) by the shift value.
  • Assume input is lowercase and alphabetic only.
Example Test Cases:
Encrypt: "attack" with shift 3 → "dwwdfn"
Decrypt: "dwwdfn" with shift 3 → "attack"
Encrypt: "hello" with shift 3 → "khoor"
Task: Encrypt a message to your partner and have them decrypt it!
MEDIUM

Vigenère Cipher

Scenario: Your spy agency needs a more secure cipher than Caesar. The Vigenère cipher uses a keyword to determine different shift values for each letter in the message, making it harder to crack.
Function Specification:
  • Define vigenere_encrypt(text, keyword) and vigenere_decrypt(text, keyword).
  • For each character in the text, use the corresponding letter in the keyword to determine the shift value.
  • If the keyword is shorter than the message, repeat the keyword.
  • Assume input is lowercase and alphabetic only.
Example:
Message: "hello"
Keyword: "key" (repeats as "keyk")
- For 'h', shift by 'k' (10) → 'r'
- For 'e', shift by 'e' (4) → 'i'
- For 'l', shift by 'y' (24) → 'j'
- For 'l', shift by 'k' (10) → 'v'
- For 'o', shift by 'e' (4) → 's'
Result: "rijvs"
Task: Implement the Vigenère cipher and exchange encrypted messages with your classmates.
HARD

Rail Fence Cipher

Scenario: Your agency needs a different type of cipher called a transposition cipher. The Rail Fence cipher (also called a zigzag cipher) writes the message in a zigzag pattern across a specified number of rows, then reads the message row by row.
Function Specification:
  • Define rail_fence_encrypt(text, rails) and rail_fence_decrypt(text, rails).
  • The "rails" parameter specifies the number of rows to use in the zigzag pattern.
  • Spaces in the original message are ignored.
Example (with 3 rails):
Original text: "WEAREDISCOVERED"
W . . . E . . . C . . . E . . .
. A . . R . . . I . . . V . . .
. . D . . . S . . . O . . . D .
Reading row by row: "WECERIVODDSAE"
Task: Implement the Rail Fence cipher and experiment with different numbers of rails.

Section 3: Advanced Challenges

MEDIUM

Password Strength Checker

Scenario: A company has hired you to implement a password strength checker for their new account system. The checker should evaluate passwords based on multiple criteria and provide a strength rating.
Function Specification:
  • Define check_password_strength(password).
  • Evaluate the password on criteria like length, use of uppercase/lowercase letters, numbers, and special characters.
  • Return a rating: "Weak", "Medium", "Strong", or "Very Strong".
Example Criteria:
- Length >= 8 characters: +1 point
- Contains both uppercase and lowercase: +1 point
- Contains at least one number: +1 point
- Contains at least one special character: +1 point

Rating:
0-1 points: "Weak"
2 points: "Medium"
3 points: "Strong"
4 points: "Very Strong"
Task: Implement the password strength checker and test it on various passwords.
HARD

Password Cracking Simulation

Scenario: You're learning about password security by simulating a dictionary attack. You have a list of common passwords and a hash function. Your goal is to "crack" a given hash by finding which password in your dictionary produced it.
Function Specification:
  • Define crack_password(target_hash, password_list, hash_function).
  • Apply the hash function to each password in the list.
  • Return the password that produces the target hash, or "Password not found" if none match.
Example:
Target hash: 528
Password list: ["password", "123456", "qwerty", "admin", "welcome"]
Hash function: ASCII sum

Result: "admin" (ASCII sum of "admin" = 97+100+109+105+110 = 521 + (length = 5) × 10% = 526)

Note: For this exercise, we'll use a simple hash function: ASCII sum + (length of password × 10%)
Task: Implement the password cracker and test it on various hash values. Discuss why real password hashing algorithms are much more complex.
